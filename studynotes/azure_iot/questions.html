Vim Questions

  1. What are the keystrokes that help exit to normal mode from insert/visual/normal mode?

    Ctrl+c, Ctrl+[, Esc.

  2. Which keystroke starts the line wise visual mode?

    Shift+v.

  3. Which keystroke starts the block wise visual mode?

    Ctrl+v or Ctrl+Alt+v.

  4. Which command opens Vim’s help?

    :help

  5. Which command quits the current window?

    :q

  6. Which command quits the current window without saving?

    :q!

  7. Which command saves or writes your changes?

    :w.

  8. Which command writes and quits the buffer window?

    :wq or :x

  9. Which command opens a file for editing?

    :e <path>

  10. How do you start searching a word within a buffer?

    /

  11. Which key finds the next occurence of the search?

    n

  12. Which key finds the previous occurence of the search?

    N

  13. When in normal mode, which key searches the word under the cursor forwards?

    *

  14. When in normal mode, which key searches the word under the cursor backswards?

    #

  15. Which key will undo your last edit?

    u

  16. Which key will redo your last change?

    Ctrl+r

    w

  18. Which key will move the cursor back to the previous word?

    b

  19. Which key will move the cursor to the beginning of the current line?

  54. What is a buffer?

    A buffer is an open file in memory.

  55. What happens when you close a buffer?

    It goes into hidden state.

  56. When you close a buffer, does it close the file too?

    No. When you close a buffer it does not close the file. The buffer just moves to hidden state.

  57. What are the three different types of buffer states?

    active, hidden, inactive

  58. What is the inactive state of a buffer?

    The buffer is not linked to any file and is not displayed and is empty.

  59. Which command displays the list of open buffers?

    :buffers

  60. Which command displays a buffer with a certain name or id?

    :buffer <ID_or_Name>

  61. Which command displays the next buffer?

    :bnext or :bn

  62. Which command displays the previous buffer?

    :bprevious or :bp

  63. Which command displays the first buffer?

    :bfirst or :bf

  64. Which command displays the last buffer?

    :blast or :bl

  65. What is an alternate buffer?

    It is indicated in your buffer list with the symbol #.

  66. Which command switches to the alternate buffer?

    :Ctrl+^

  67. Which command helps switch to a specific buffer ID?

    <ID>Ctrl+^

  68. Which command will you use to switch to a buffer with an ID of 75?

    75Ctrl+^

  69. Which command helps apply a command to all buffers?

    :bufdo <command>

  70. Which command displays a list of unlisted buffers?

    :buffers! or :ls!

  71. How do you create a new file in a buffer?

    :badd <filename>

  72. How do you delete a buffer with an ID or name?

    :bdelete <ID_or_Name>

  73. Write a command that will delete a buffers from ID 1 to ID 10?

    1,10bdelete

  74. Write a command that will delete all buffers?

    %bdelete

  75. Which help command will display information about buffers?

    :help buffers and :help :buffers

  147. What are quickfix lists?

    A quickfix list is a set of positions in one or multiple files.

  148. What happens when you run the command :vimgrep hello *?

    1. It will search the pattern "hello" in every file of your working directory.

    2. It will populate a quickfix list with every position matching your pattern "hello".

    3. It will move your cursor to the first position of the quickfix list.

  149. What is a quickfix list often used for?

    The quickfix list is often used to display specific errors in a codebase, often spit out from a compiler or a linter.

  150. Which command will display the current quickfix list?

    :cl or :clist

  151. Which command will move to the <number> entry of the current quickfix list?

  152. Which command will move to the next entry of the current quickfix list?

  155. Which command will move to the last entry of the current quickfix list?

  156. How will you execute a command <cmd> on each valid entry of the current quickfix list?

    :cdo <cmd>

  157. How can you create a quickfix list using the result of a Vimscript extension <expr>?

    :cexpr <expr> or :cex <expr>

  158. Which command will empty your current quickfix list?

    :cex []

  159. Which command will populate your quickfix list with the output of any shell command like ls?

    :cex system("ls")

  160. How can you append the result of a vimscript expression to the current quickfix list?

    :caddexpr <expr> or :cadde <expr>

  161. Which command opens the current quickfix list in a new buffer window?

  162. What is a location list?

    A location list is similar to a quickfix list, except that a location list is local to a window and a quickfix list is global to your Vim instance.
    You can have one location list per open window.
    You can have access to only one quickfix list.

  163. What is a register??

    Registers are places where you can read and write some text.
    Think of them as Vim’s clipboard.

  164. How will you display the contents of a register?

    :registers or :reg

  165. Which keystroke specifies the register <reg> to be read or written?

    "<reg>

  166. How will you write to a register?

    To write to a register:

    • Hit "a in NORMAL mode to specify what register you want to write on.

    • Yank, change or delete some content to write it to register a.

      • for example, by using dd in normal mode.

  167. How will you read from a register a?

    • Hit "a in NORMAL mode to specify what register you want to read.

    • Use a put keystroke in NORMAL mode to spit out the content of the register in your current buffer.

      • An example of a put keystroke is a P or p command.

  168. How many types of registers are there in Vim?

    There are 10 different types of registers.

  169. What does the unnamed register contain?

    It contains the last deleted, changed, or yanked content even if one register is specified.

  170. How will you paste content present in the unnamed register?

    ""p

  171. What does the 0 numbered register contain?

    The content of the last yank.

  172. How will you paste content present in the last yank using a register?

    "0

  173. What do the numbered registers 1 to 9 contain?

    They contain the content you have deleted or changed.

    1. Each time you delete or change content, it will be added to register 1.

    2. The previous content of the register 1 will be assigned to register 2, the previous content of 2 to 3…​

    3. When something is added to the register 1, the content of the register 9 is lost.

      None of these registers are written if you’ve specified one before with the keystroke ".

  174. What is the small delete register?

    The - register.
    It contains any deleted or changed content smaller than one line.
    It is not written if you specified a register with ".

  175. What are named registers?

    They range from a to z.
    Vim will never write to them if you do not specify the keystroke ".
    You can use the uppercase name of each register to append to it (instead of over-writing it).

  176. What are read only registers?

    They are "., "% and ":.

    • . contains the last inserted text.

    • % contains the name of the current file.

    • : contains the most recent command line executed.

  177. What is an alternate buffer register?

    It is #.
    It contains the alternate buffer for the current window.

  178. What is an expression register?

    It is "=. It stores the result of an expression.

  179. What are the selection registers?

    They are "+ and "*.

    • + is synchronized with the system clipboard.

    • * is synchronized with the selection clipboard (only on *nix systems).

  180. What is a black hole register?

    It is the "_ register.
    Everything written in there will disappear forever.

  181. What is the last search pattern register?

    This register contains your last search.
    It is the "/ register.

  182. What gets output when the put command is used without specifying the register?

    It will output the unnamed register by default.

  183. What happens when you add the line clipboard+=unnamedplus to your vimrc?

    In this case, the content you change, delete, or yank will go directly in the unnamed register and the + register.

Using the put commands will directly output the + register too.

Many find it useful to access the OS clipboard more easily, without the need to specify the + register for reading and writing it.

How can you append to your macro recording?

  1. Hit qa and record whatever keystrokes you want.

  2. Stop the recording by hitting q again.

  3. You realize that you forgot a couple of keystrokes.

  4. Execute your keystrokes to be sure you are at the last position of your recording.

  5. Hit qA to append to your register a.

  6. When you are done, hit q again.

How can you paste or use the content of a register in INSERT mode or COMMAND LINE mode?

Ctrl+R<reg> to put the content of <reg> in your current buffer.

What is an expression register?

It can evaluate custom functions that you have defined while staying in insert mode.

Give an example of how you can use an expression register?

  1. Switch to INSERT mode and hit keystroke Ctrl+r =.

  2. You will move to the Vim’s command line.

  3. Type any vimscript expression you want, like system("ls") or 4+4.

  4. Hit Enter to run the expression.

  5. The output of the shell command ls is directly inserted in the buffer.

How can you clear a register?

qaq
Beginning a recording also deletes everything which is in this register.
So you need to just stop the recording by hitting q again to have an empty register.

What is a substitution?

A substitution is a way to replace some content with some other content, using a range and a count.
The count will decide how many lines is affected by your substitution from the last line of your range.

What is the default range of a substitution command?

The default range is the current line if you do not specify it.

What is the pattern of the substitution command?

:s/pattern/replacement/flags

  • pattern is the search you want to match.

  • replacement will replace the first match of the pattern on each line.

  • flag modify the behaviour of the command.

Is the replacement mandatory in a substitution command?

No, if you omit it, the substitute command will delete the pattern matched.
